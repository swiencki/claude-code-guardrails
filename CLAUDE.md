## Project Overview

This repo provides a modular, composable guardrail system for Claude Code. It organizes guardrails into six layers under `layers/`, with a build script that merges selected layers into `.claude/settings.json`.

## Build System

- `make build` merges layers into `.claude/settings.json`
- `make test` runs the test suite (46 tests)
- The build script is `scripts/build-settings.sh` - the Makefile is a thin wrapper
- `.claude/settings.json` is a generated file - edit the layer fragments in `layers/` instead

## Rules

- Never edit `.claude/settings.json` directly - it is generated by `make build`
- Always run `make test` after changing layer fragments or the build script
- Hook fragments in `layers/2-hooks/` must follow the PreToolUse schema with matcher, hooks array, type, command, and statusMessage fields
- Permission fragments in `layers/3-permissions/` must have `permissions.allow` and `permissions.deny` arrays
- All fragments must include a top-level `description` field
- Keep each hook concern in its own file (one file per category, not one giant file)
